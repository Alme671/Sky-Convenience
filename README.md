# ⚡ GBF Auto Reflash

> Granblue Fantasy 自动刷新辅助 — 检测到指定请求后随机延迟自动后退

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Tampermonkey](https://img.shields.io/badge/Tampermonkey-Ready-green.svg)](https://www.tampermonkey.net/)

---

## ✨ 功能特性

- 🔄 **自动后退** — 检测到匹配的网络请求后，自动触发浏览器后退
- ⏱️ **随机延迟** — 在可配置的时间范围内随机等待，模拟人工操作
- ⚔️ **普攻结算检测** — 内置 `normal_attack_result.json` 匹配，一键开关
- ✏️ **自定义匹配** — 支持自定义 URL 匹配模式，逗号或换行分隔多个
- 🎛️ **快捷预设** — 内置「关闭」「日常」「猎金」三种一键预设，通过 Tampermonkey 菜单快速切换
- 🖱️ **自动点击** — 后退后自动在游戏区域随机位置点击，恢复自动攻击，无需手动操作
- ⏰ **空闲检测** — 启用后长时间未检测到匹配请求时发送桌面通知，提醒可能遇到验证码或战斗结束
- 🔗 **仅路径匹配** — 自动忽略 `?_=...&t=...&uid=...` 等查询参数
- ❄️ **冷却保护** — 可配置冷却时间，防止短时间内重复触发
- 📋 **请求日志** — 面板内查看拦截到的请求，无需打开 F12
- 🌐 **全面拦截** — 同时拦截 XHR 和 Fetch 请求
- 📌 **控制面板** — 可拖拽、可最小化的浮动面板（默认隐藏，菜单开启）
- 💾 **配置持久化** — 所有设置自动保存，刷新页面后自动恢复
- 🔔 **桌面通知** — 触发时可选显示桌面通知

## 📦 安装

### 前置要求

- 浏览器已安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展

### 安装步骤

1. 点击 [安装脚本](gbf_autoReflash.user.js)（或手动打开 `gbf_autoReflash.user.js`）
2. Tampermonkey 会自动识别脚本并弹出安装页面
3. 点击 **安装** 即可

## 🚀 使用方法

### 快速开始

1. 打开 GBF 游戏页面
2. 右键点击浏览器右上角的 **Tampermonkey 图标**
3. 选择「日常」或「猎金」预设，一键启用
4. 开始游戏，脚本会自动检测、后退、并点击屏幕恢复攻击

### Tampermonkey 菜单

右键 Tampermonkey 图标可看到以下菜单：

| 菜单项 | 效果 |
|--------|------|
| **关闭 — Close** | 关闭所有自动后退功能 |
| **日常 — 1-2s 自动后退** | 日常刷本模式 |
| **猎金 — 快速 0.3-1s** | 猎金快速模式（普攻+技能+召唤） |
| **📌 显示/隐藏面板** | 需要微调配置时打开面板 |

### 快捷预设

| 预设 | 监听端点 | 延迟 | 冷却 |
|------|----------|------|------|
| 日常 | `normal_attack_result.json` | 1-2s | 4s |
| 猎金 | `normal_attack_result.json` + `ability_result.json` + `summon_result.json` | 0.3-1s | 0.3s |
| 关闭 | — | — | — |

### 匹配 URL 说明

脚本会检测类似以下格式的请求：

```
https://game.granbluefantasy.jp/rest/multiraid/normal_attack_result.json?_=1775887819343&t=1775887819344&uid=30744529
https://game.granbluefantasy.jp/rest/multiraid/ability_result.json?_=1775891954396&t=1775891954396&uid=30744529
https://game.granbluefantasy.jp/rest/multiraid/summon_result.json?_=1775891946689&t=1775891946689&uid=30744529
```

开启 **仅匹配路径** 后（默认开启），会忽略 `?` 后的参数，只匹配路径部分。

### 自动点击

后退后游戏会出现约 500ms 黑屏，需要点击屏幕才能恢复自动攻击。脚本会在后退后随机 600-1500ms 内自动模拟点击：

- **随机坐标**：点击位置在游戏画面中心区域（水平 40%-60%，垂直 30%-50%）随机偏移
- **完整事件序列**：`mousedown → mouseup → click`，模拟真实鼠标行为
- **默认开启**，可通过面板关闭或调整延迟范围

### 空闲检测

当脚本处于启用状态时，如果超过设定时间（默认 60 秒）未检测到任何匹配请求，会自动发送桌面通知提醒：

- 可能遇到了**验证码**需要手动处理
- 可能**战斗已结束**需要开始新的战斗
- 设为 `0` 可关闭此功能

### 配置说明

| 选项 | 说明 | 默认值 |
|------|------|--------|
| 启用自动后退 | 主开关 | 关闭 |
| 普攻结算 | 内置 `normal_attack_result.json` 匹配 | 开启 |
| 自定义匹配 | 自定义 URL 匹配模式，逗号或换行分隔多个 | 空 |
| 使用正则匹配 | URL 匹配使用正则表达式 | 关闭 |
| 仅匹配路径 | 只匹配 URL 路径，忽略查询参数 | 开启 |
| 最小延迟 (ms) | 触发后退前等待的最短时间 | 1000 |
| 最大延迟 (ms) | 触发后退前等待的最长时间 | 3000 |
| 冷却时间 (ms) | 两次触发之间的最短间隔 | 2000 |
| 后退后自动点击 | 后退后模拟点击恢复自动攻击 | **开启** |
| 点击最小延迟 (ms) | 后退后等待点击的最短时间 | 600 |
| 点击最大延迟 (ms) | 后退后等待点击的最长时间 | 1500 |
| 空闲超时通知 (ms) | 长时间无匹配请求时通知，0=关闭 | 60000 |
| 桌面通知 | 触发时显示系统桌面通知 | 关闭 |
| 调试模式 | 记录所有拦截到的请求到面板日志 | 关闭 |

### 面板选项卡

| 选项卡 | 功能 |
|--------|------|
| ⚙️ 配置 | 主开关、预设、匹配模式、延迟设置、自动点击、空闲检测 |
| 📋 日志 | 实时查看拦截到的请求记录 |

> 面板当前已禁用以提升脚本性能。如需重新启用，请取消脚本中 `UI 面板` 区域的块注释，并恢复 `init()` 中的 `createPanel()` 调用。

### 使用示例

**场景一：日常刷本**

右键 Tampermonkey → 选择「日常」→ 自动开启普攻结算检测 → 后退后自动点击屏幕继续攻击

**场景二：猎金速刷**

右键 Tampermonkey → 选择「猎金」→ 同时监听普攻/技能/召唤结算 → 快速后退并自动恢复

**场景三：使用自定义匹配**

1. 通过菜单打开面板，开启 **🐛 调试模式**
2. 进行一次正常游戏操作，切换到 **📋 日志** 查看拦截到的请求
3. 找到目标 URL，复制关键片段粘贴到 **✏️ 自定义匹配** 输入框
4. 开启自动后退，关闭调试模式

### 技术实现

```
网络请求 → XHR/Fetch 拦截器 → 路径提取 → 多模式匹配 → 冷却检查 → 随机延迟 → history.back() → 随机延迟 → 模拟点击 → 空闲计时器重置
```

关键设计：
- **`@run-at document-start`**：在文档加载前注入拦截器，确保不遗漏任何请求
- **XHR + Fetch 双拦截**：覆盖所有现代浏览器的网络请求方式
- **路径提取 (pathOnly)**：使用 `URL` API 解析，仅比较路径部分，避免时间戳等参数干扰
- **冷却机制**：防止同一次操作触发多个 result 请求时重复后退
- **定时器去重**：同一时间只保留一个挂起的后退定时器
- **人类模拟点击**：随机坐标 + 完整 MouseEvent 序列，降低自动化检测风险
- **空闲检测**：验证码/战斗结束时通过桌面通知提醒用户
